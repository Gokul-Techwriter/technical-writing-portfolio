## Introduction

Postman is an API platform developed by Postman Inc. This guide helps users navigate the Postman interface, including Collections, Environments, Requests, and API testing features on Windows.

## How to Use This Guide

Read this guide from top to bottom if you are a beginner. If you are looking for a specific feature, use **Ctrl+F** to search by keyword or jump to the relevant section using the Table of Contents.

## Prerequisites

Before you begin, ensure the following requirements are met:

- [ ] A stable internet connection.  

- [ ] Free Postman account created.

- [ ] Postman is installed on your device.  
Download: https://www.postman.com/downloads/

## Getting Started

### Step 1: Launch Postman App

Double-click the **Postman icon** on your desktop or search the app name on  **Start menu** and click **Open**.

### Step 2: Explore the Welcome Screen

Go through the Sign-up procedure, elements, icons, and the instructions displayed on the screen after opening the app for the first time.

### Step 3: Understand the Interface

There are 5 main elements on the left sidebar:

- **Collections:** It enables users to save all the API requests, keep them organized, and share them with team members or anyone.

- **Environments:** It allows users to set variables that can be adjusted based on the work context in Postman.

- **Spec Hub:** It enables users to design their API in an OpenAPI 3.0 or AsyncAPI 2.0 specification.

- **Flows:** It is a visual, low-code tool designed to make API workflows faster and easier.

- **Mock servers:** They allow users to imitate an API for testing before setting up a real API server.


> 💡 **Tip:** You can also use a Google account/GitHub account, and you can do Single Sign-On to create an account.

> ⚠️ **Warning:** Read and ensure the Terms and Privacy policy. If not, unexpected issues may arise.

## Key Features

### Feature 1: Terminal

Terminal allows running commands within the Postman app, eliminating the use of an external application for it.

### Feature 2: Integration

Integration lets different applications work together with the Postman app. Its services will be available right in the app, so you will not need to access them outside.

### Feature 3: Monitors

Monitors enable continuous checking of the health and performance of the APIs.

### Feature 4: Agent Mode

Agent Mode helps to complete tasks faster across collections, tests, workspaces, environments, monitors, and Spec Hub. It can help create and manage requests, flows, and mock servers, debug errors, and write tests.

> 💡 **Tip:**  Ensure the app is of the latest version for the best performance of this mode.

## Common Tasks

### Task 1: Account Creation

> Note: The first interface that comes after launching the app for the first time is the account creation process.

**Steps:**

1. Enter your email address, and click **Create Free Account**.

2. Set a name and password for your account > click **Create Free Account**.

3. Verify the email by using the code sent to it.

4. Click **Allow access** in the **window security alert tab**.

5. Enter your name and role for the workspace; write the request in the **request box**.

6. Click **Get started with AI**.

> 💡 **Tip:** Enter “Sent a GET request” in the request box for a start if you are confused.

**Expected Result**: Account is created successfully without any errors.

### Task 2: Create a New Request

**Steps:**

1. Click **Create New** (**Ctrl+N**) or click on the **‘+’ icon** to create a new request.

2. Choose **Save**

3. Enter the title and select the **collection** to which you want to save the request.

4. Click **Save**

**Expected Result:** The new request file is created successfully without any errors.

### Task 3: Change the Color Theme

**Steps:**

1. Click on the **Settings icon** > **App settings**.

2. Choose **Themes**

3. Select the theme for daytime.

4. Select the theme for nighttime.

> 💡 **Tip:** Changing the color theme to dark is easier on the eyes during long sessions.

**Expected Result:** Color theme is changed successfully.

### Task 4: Change Font Size

**Steps:**

1. Open **Settings** > **App settings**

2. Choose **General** > scroll down to **Editor settings**.

3. Adjust the **Font size (px)** according to your preference.

> 💡 **Tip:**  Always choose a font size that is comfortable to read for both you and your readers.

**Expected Result:** Font size is changed successfully to the preferred size.

### Task 5: Rename Workspace

**Steps:**

1. Open **Settings** > **Workspace settings**

2. Click on the **Workspace name**  

3. Give a new name, and tap anywhere to save.

**Expected Result:** The Workspace name is changed successfully without any errors.

### Task 6: Delete a Workspace

**Steps:**

1. Open **Settings** > **Workspace settings**

2. Scroll down to the bottom > click **Delete workspace**.

3. Enter **Delete** in the box > click **Delete workspace**.

> ⚠️ **Warning:** Once a workspace gets deleted, there is no way to recover the workspace and its data.

**Expected Result:** The Workspace is deleted successfully.

## Troubleshooting

### Issue 1: Account creation failed

 **Cause:**

- Unstable internet connection

- Invalid email ID or password

- Wrong verification code

**Fix:**

1. Connect your system to a stable internet.

2. Confirm that the email ID and its password you entered are correct.

3. Check that you receive the latest verification code from the Postman mail account.

4. Repeat the action again.

**Expected Result:** The account is created successfully without any errors.

### Issue 2: App closing automatically

**Cause:**

- Outdated version of the app

- Corrupted installer file

**Fix:**

1. Ensure that the app is of the latest version. Go to **Menu icon** > **Help** > **Check for Updates**.

2. Download the latest installer file from the official website and install it.

**Expected Result:** The automatically closing issue is resolved successfully.

### Issue 3: Unable to Send a Request

**Cause:**

- Unstable internet connection

- Invalid or incorrect API URL

- Incorrect request method selected 

**Fix:**

1. Ensure your system has a stable internet connection.

2. Verify the API URL is correct and properly formatted.

3. Confirm you have selected the correct request method from the dropdown menu.

4. Click **Send** again to retry the request.

**Expected Result:** The request is sent successfully and a response is received without any errors.

## FAQ

**Q1: What is Postman?**

**A:** Postman is a unified platform for designing, testing, distributing, documenting, and monitoring APIs. You can create teams to work together on API development. 

**Q2: Is the Postman app available on any mobile device?**

**A:** No, the Postman app is not officially available on mobile devices, but you can access the web version on mobile.

**Q3: I accidentally deleted my Workspace. Is there any way to recover it?**

**A:** No, once a workspace gets deleted, there is no way to recover the workspace and its data.

**Q4: How can I access the integration option?**

**A:** Postman provides integration options for accessing that:

1. Click on the **Home icon** in the top left of the screen.

2. Choose **Integration** > Browse All Integration.

3. Select the integration you want > click **Add Integration**.

**Q5: How do I clear the cache ?**

**A:** You can clear the cache by going to the **Menu icon** in the top left corner > Choose **Help** > click **Clear cache and reload**.
